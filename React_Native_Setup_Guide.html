3. Complete Installation Guide

Step 1: Enable PowerShell Script Execution

Purpose: Allow PowerShell to run scripts on your system.

Open PowerShell as Administrator (Right-click Start ‚Üí Windows PowerShell (Admin))

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

When prompted, type A (Yes to All) and press Enter.

Step 2: Install Chocolatey Package Manager (Optional but Recommended)

Purpose: Simplifies installation of development tools.

Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

Verify installation:

choco --version

Step 3: Install Node.js and npm

Purpose: JavaScript runtime required for React Native.

Method 1: Using Chocolatey

choco install nodejs -y

Method 2: Manual Download

Download from https://nodejs.org/ (Choose LTS version)

Verify installation:

node --version
npm --version
Expected: Node.js v18.x.x or higher, npm v9.x.x or higher

Step 4: Install Yarn Package Manager

Purpose: Alternative package manager used by this project.

npm install -g yarn

Verify installation:

yarn --version

Step 5: Install Java Development Kit (JDK) 17

Purpose: Required for Android app compilation.

Important: Must be JDK 17, not JDK 11 or JDK 18+

Using Windows Package Manager (winget):

winget install EclipseAdoptium.Temurin.17.JDK

Installation will show progress and complete automatically (~200MB download)

Verify installation:

java -version
Expected output: openjdk version "17.0.x"

Step 6: Install Android Studio

Purpose: Provides Android SDK and development tools.

winget install Google.AndroidStudio
Note: This is a large download (~1.4GB) and may take 10-30 minutes.

The installer will request administrator privileges - approve when prompted.

Step 7: Set Up Android SDK

Purpose: Install Android development tools and SDK packages.

7.1 Create SDK Directory:

New-Item -ItemType Directory -Path "$env:USERPROFILE\AppData\Local\Android\Sdk" -Force

7.2 Download Command Line Tools:

$sdkPath = "$env:USERPROFILE\AppData\Local\Android\Sdk"
$sdkUrl = "https://dl.google.com/android/repository/commandlinetools-win-11076708_latest.zip"
Invoke-WebRequest -Uri $sdkUrl -OutFile "$sdkPath\commandlinetools.zip"

7.3 Extract and Organize Tools:

Expand-Archive -Path "$sdkPath\commandlinetools.zip" -DestinationPath "$sdkPath\temp" -Force
New-Item -ItemType Directory -Path "$sdkPath\cmdline-tools\latest" -Force
Get-ChildItem -Path "$sdkPath\temp\cmdline-tools" | Move-Item -Destination "$sdkPath\cmdline-tools\latest" -Force
Remove-Item "$sdkPath\temp" -Recurse -Force
Remove-Item "$sdkPath\commandlinetools.zip"

7.4 Set Environment Variables:

$env:ANDROID_HOME = "$env:USERPROFILE\AppData\Local\Android\Sdk"
$env:PATH = "$env:ANDROID_HOME\cmdline-tools\latest\bin;$env:ANDROID_HOME\platform-tools;$env:PATH"

7.5 Install SDK Packages:

echo "y" | sdkmanager "platforms;android-34" "platforms;android-35" "build-tools;34.0.0" "build-tools;35.0.0" "platform-tools" "ndk;26.1.10909125"
This downloads approximately 2GB of SDK components and may take 15-30 minutes.

Step 8: Install Android Debug Bridge (ADB)

Purpose: Communicate with Android devices.

winget install Google.PlatformTools

Verify installation:

adb --version

4. Project Setup & Configuration

Step 1: Navigate to Project Directory

cd C:\Projects\UmairProject\Last_Mile_Delivery

Step 2: Install Project Dependencies

Purpose: Download all required npm packages.

yarn install
This downloads 500MB-1GB of packages and may take 5-10 minutes.

Step 3: Fix Dependency Compatibility Issues

Purpose: Downgrade incompatible packages.

yarn add react-native-reanimated@3.6.0

Step 4: Create Configuration Files

4.1 Create react-native.config.js in project root:

@"
module.exports = {
  dependencies: {
    'react-native-reanimated': {
      platforms: {
        android: null,
      },
    },
    'react-native-gesture-handler': {
      platforms: {
        android: null,
      },
    },
  },
};
"@ | Out-File -FilePath "react-native.config.js" -Encoding UTF8

4.2 Create local.properties in android folder:

cd android
@"
sdk.dir=$env:USERPROFILE\\AppData\\Local\\Android\\Sdk
"@ | Out-File -FilePath "local.properties" -Encoding UTF8

Step 5: Update Android Configuration Files

5.1 Update android/gradle.properties

Add these lines at the end of the file:

# Optimization settings
android.suppressUnsupportedCompileSdk=35
org.gradle.parallel=false
org.gradle.configureondemand=false
org.gradle.daemon=false
org.gradle.caching=false
org.gradle.jvmargs=-Xmx4096m -XX:MaxMetaspaceSize=1g -XX:+HeapDumpOnOutOfMemoryError -Dfile.encoding=UTF-8
org.gradle.workers.max=1
android.enableJetifier=true
org.gradle.java.home=C:\\Program Files\\Eclipse Adoptium\\jdk-17.0.16.8-hotspot

5.2 Update android/build.gradle

Modify the buildscript section:

buildscript {
    ext {
        buildToolsVersion = "35.0.0"
        minSdkVersion = 23
        compileSdkVersion = 35
        targetSdkVersion = 34
        ndkVersion = "26.1.10909125"
        kotlinVersion = "1.9.24"
    }
    dependencies {
        classpath("com.android.tools.build:gradle:8.6.0")
        classpath("com.facebook.react:react-native-gradle-plugin")
        classpath("org.jetbrains.kotlin:kotlin-gradle-plugin")
    }
}

5.3 Update android/gradle/wrapper/gradle-wrapper.properties

Change the distribution URL:

distributionUrl=https\://services.gradle.org/distributions/gradle-8.9-all.zip

Step 6: Create Build Scripts

Create build-android.bat in android folder:

@echo off
echo Killing all Java processes...
taskkill /F /IM java.exe 2>nul
taskkill /F /IM javaw.exe 2>nul

echo Setting environment...
set JAVA_HOME=C:\Program Files\Eclipse Adoptium\jdk-17.0.16.8-hotspot
set ANDROID_HOME=%USERPROFILE%\AppData\Local\Android\Sdk
set PATH=%JAVA_HOME%\bin;%ANDROID_HOME%\platform-tools;%PATH%
set GRADLE_OPTS=-Xmx2048m -Dorg.gradle.daemon=false -Dorg.gradle.parallel=false -Dorg.gradle.workers.max=1

echo Cleaning cache...
rd /s /q .gradle 2>nul
rd /s /q app\build 2>nul

echo Building APK (this will take time, please be patient)...
call gradlew.bat assembleDebug --no-daemon --no-parallel --no-build-cache --max-workers=1 -Dorg.gradle.jvmargs="-Xmx2048m" --warning-mode=none

if %errorlevel% == 0 (
    echo.
    echo BUILD SUCCESSFUL!
    echo APK Location: app\build\outputs\apk\debug\app-debug.apk
) else (
    echo.
    echo BUILD FAILED!
)

pause

5. Building the Application

Method 1: PowerShell Script (Recommended)

Purpose: Uses optimized settings to prevent build hanging and memory issues.

Use the working PowerShell script (most reliable):

.\build-android-working.ps1
This script automatically: ‚Ä¢ Sets environment variables ‚Ä¢ Kills stuck processes ‚Ä¢ Builds with optimal settings ‚Ä¢ Auto-installs if device detected

Method 2: Optimized Script (For Build Issues)

If build-android-working.ps1 fails or hangs:

.\build-android-optimized.ps1

This uses even more conservative settings to prevent hanging.

Method 3: Manual Build

Navigate to android folder and run:

cd android
.\gradlew --stop
.\gradlew clean --no-daemon
.\gradlew assembleDebug --no-daemon --max-workers=1 --no-parallel --console=plain
Expected Build Times:
‚Ä¢ First build: 30-60 minutes
‚Ä¢ Subsequent builds: 5-15 minutes
‚Ä¢ After clean: 15-30 minutes
Build Output:
APK Location: android\app\build\outputs\apk\debug\app-debug.apk
Expected Size: ~150-160 MB

6. Running the Application

Multiple Methods Available: Choose the method that works best for your setup.

Method 1: React Native CLI (Recommended)

This is the fastest method once everything is set up:

cd C:\Projects\UmairProject\Last_Mile_Delivery
npx react-native run-android
This command automatically: ‚Ä¢ Starts Metro bundler ‚Ä¢ Builds the APK if needed ‚Ä¢ Installs on connected device ‚Ä¢ Launches the app

Method 2: Using Yarn Scripts

Alternative React Native CLI method:

cd C:\Projects\UmairProject\Last_Mile_Delivery
yarn android

This runs the same as npx react-native run-android

Method 3: Manual APK Install + Metro

Use this if the above methods fail:

Step 1: Build APK using PowerShell script:

.\build-android-working.ps1

Step 2: Start Metro in separate terminal:

cd C:\Projects\UmairProject\Last_Mile_Delivery
yarn start

Step 3: Launch app on device (find "ReactProject" icon)

Prerequisites: Prepare Your Android Device

1.1 Enable Developer Options:

  • Go to Settings ‚Üí About Phone
  • Tap "Build Number" 7 times
  • Enter your PIN/Password if prompted

1.2 Enable USB Debugging:

  • Go to Settings ‚Üí Developer Options
  • Enable "USB Debugging"
  • Enable "Install via USB" (if available)

1.3 Connect Device:

  • Connect phone via USB cable
  • When prompted on phone, tap "Allow" for USB debugging
  • Check "Always allow from this computer"

1.4 Verify Connection:

adb devices
Should show your device ID with "device" status

Step 2: Install the APK

cd android
adb install app\build\outputs\apk\debug\app-debug.apk
Expected: "Success" message after installation

Step 3: Start Metro Bundler

In a new PowerShell window:

cd C:\Projects\UmairProject\Last_Mile_Delivery
yarn start
Keep this window open while running the app

Step 4: Launch the Application

Find and tap the "ReactProject" app icon on your phone

The app should connect to Metro bundler automatically

7. Troubleshooting Guide

Issue Solution
Build stuck at "Configure project :react-native-reanimated" 1. Press Ctrl+C to stop
2. Run: taskkill /F /IM java.exe
3. Use the build-android.bat script
SDK location not found Create local.properties:
echo "sdk.dir=%USERPROFILE%\AppData\Local\Android\Sdk" > android\local.properties
JDK version issues Set JAVA_HOME:
$env:JAVA_HOME = "C:\Program Files\Eclipse Adoptium\jdk-17.0.16.8-hotspot"
Out of memory error Increase memory:
$env:GRADLE_OPTS = "-Xmx4096m -XX:MaxMetaspaceSize=1g"
Device not detected 1. Check USB cable
2. Reinstall device drivers
3. Try different USB port
4. Restart ADB: adb kill-server && adb start-server
Metro bundler connection failed 1. Check firewall settings
2. Run: adb reverse tcp:8081 tcp:8081
3. Shake device and select "Reload"
